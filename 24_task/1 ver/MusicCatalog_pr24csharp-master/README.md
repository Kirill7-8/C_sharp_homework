Проект реализован на основе **Чистой архитектуры** (Clean Architecture) с разделением на четыре основных слоя: **Domain** (ядро — сущности Disc, Song с бизнес-правилами и валидацией), **Application** (use cases — команды, запросы и хендлеры, а также порты-интерфейсы репозиториев), **Infrastructure** (реализация репозиториев с JSON-сериализацией для хранения данных в файлах) и **Presentation** (Web API с Swagger для тестирования и консольное приложение для локального использования). Проект следует принципам DDD (богатая модель, инкапсуляция), CQRS (разделение команд и запросов) и Ports & Adapters (интерфейсы в Application, реализация в Infrastructure). 


**Вариант 8**: Реализован каталог музыкальных компакт-дисков, который позволяет: добавлять и удалять диски; добавлять и удалять песни на диске; просматривать содержимое целого каталога и каждого диска в отдельности; осуществлять поиск всех песен заданного исполнителя по всему каталогу; сортировать песни на каждом диске в алфавитном порядке названий песен или фамилий исполнителей.
